1. 项目概述
1.1 项目简介
本项目是一款面向微信小游戏平台的塔防+Roguelike（肉鸽）类型的休闲策略游戏。玩家在随机生成的关卡地图上建造防御塔，抵御一波波敌人的进攻。每局游戏关卡随机、天赋随机、敌人组合随机，带来高重复可玩性。游戏采用竖屏单手操作，适配微信小游戏碎片化游玩场景。
1.2 核心卖点
塔防策略 + Roguelike随机性的深度融合
单局时长 5~15 分钟，适合碎片化时间
丰富的Build组合，每局体验不同
竖屏单手操作，微信内即点即玩
像素/Q版卡通美术风格，轻量包体

3. 核心玩法设计
3.1 核心循环
游戏核心循环为：① 进入随机生成关卡 → ② 布置防御塔/选择路线 → ③ 抵御敌人波次 → ④ 战斗中随机获得技能/资源 → ⑤ 波次间升级/调整 → ⑥ Boss战 → ⑦ 获得奖励/解锁内容 → ⑧ 回到①
3.2 战斗机制
自动战斗 + 玩家主动释放技能：防御塔自动攻击，玩家可点击释放主动技能
能量系统：击杀敌人积累能量，能量满可释放英雄技能
塔的建造与升级：消耗金币建造/升级防御塔，升级有分支选择
手动加速：支持1x/2x/3x速度，满足不同玩家节奏需求
暂停/减速：Boss战可手动暂停调整策略
3.3 战斗视角与操作
采用2.5D俯视角，地图为网格化设计。竖屏操作，底部为建造栏和技能栏，顶部为波次信息和资源显示。支持拖拽建造、点击升级、滑动查看地图。
4. 肉鸽（Roguelike）系统
4.1 随机地图生成
每局游戏的地图由程序随机生成，包含：• 路径节点：敌人行进路线，每条路线有不同难度和奖励• 建造格：玩家可放置防御塔的位置，分布不均匀增加策略性• 事件格：随机事件（商人、宝箱、精英怪等）• Boss节点：每章节终点的Boss战
4.2 随机事件系统
每波战斗结束后，玩家从3个随机事件中选择1个：
4.3 永久成长（Meta Progression）
即使单局失败，玩家也能获得以下永久成长资源：• 灵魂碎片：解锁新防御塔、新英雄• 天赋点：在天赋树中点亮永久属性加成• 图鉴进度：解锁怪物/道具图鉴，获得收藏奖励• 段位积分：赛季排行榜依据
4.4 难度递进（Shrine系统）
通关基础难度后解锁"挑战祭坛"，玩家可主动选择叠加挑战条件（如：敌人移速+30%、塔造价+50%等），每叠加一个挑战条件，获得的奖励倍率提升。挑战越高，排名越靠前。
5. 塔防系统
5.1 防御塔分类
5.2 防御塔升级体系
每座塔有3个等级，3级时可从2个分支中选择1个进阶：• 分支A：强化现有能力（如弓箭塔→穿透箭塔/连射箭塔）• 分支B：获得全新机制（如弓箭塔→追踪箭塔/毒箭塔）进阶选择不可逆，增加策略深度。
5.3 塔的协同效果
特定塔组合放置在相邻格子时触发协同效果：• 火焰塔+毒沼塔 = "毒火"：持续燃烧伤害翻倍• 冰霜塔+雷电塔 = "超导"：减速效果提升50%• 号角塔+弓箭塔 = "精准号令"：弓箭塔暴击率提升协同效果需要玩家在图鉴中探索发现。

6.2 英雄等级与解锁
• 初始仅开放"指挥官"，其余英雄通过关卡成就/消耗灵魂碎片解锁• 英雄有独立等级（1~30级），升级解锁更强技能效果和被动天赋• 英雄等级通过游戏内经验值提升，不依赖付费
7. 关卡与地图设计
7.1 章节结构
7.2 地图生成规则
每张地图有2~4条敌人行进路径，路径之间有交叉点
建造格分布在路径两侧和交叉区域，密度不均匀
地图有高地/低地区域，高地建造的塔射程+1但造价+20%
每章有3~5种地图模板，随机抽取并微调地形
精英波次和Boss波次的路径可能发生变化（如Boss会拆墙开新路）
8. 敌人与波次设计
8.1 敌人类型
8.2 波次设计规则
每章前几波为教学/热身，难度逐渐提升
每3~5波出现一次精英波次（特殊敌人组合）
精英波次有独立奖励（额外金币/技能选择）
最后一波为Boss波次，Boss有独立技能和阶段机制
敌人属性随波次递增（生命值、攻击力、移速）
8.3 Boss设计
每个Boss有3个阶段，阶段转换触发特殊机制：• 阶段1（100%~70%血量）：常规攻击模式• 阶段2（70%~30%血量）：强化攻击+召唤小怪• 阶段3（30%以下）：狂暴模式，攻击频率翻倍，新增技能
9. 技能与法术系统
9.1 技能分类
9.2 技能品质与合成
技能有4个品质等级：白色（普通）→ 绿色（精良）→ 蓝色（稀有）→ 橙色（传说）• 低品质技能可通过合成升级为高品质技能• 合成规则：3个同品质同类型技能 → 1个高一品质技能• 传说技能有独特效果，不可合成获得，仅通过事件/Boss掉落
9.3 技能上限与管理
• 玩家最多同时携带6个主动技能 + 4个被动技能• 获得新技能时可选择替换已有技能• 技能槽位通过天赋树解锁额外数量
10. 道具与装备系统
10.1 消耗道具
10.2 遗物系统（被动装备）
遗物是单局内获得的被动装备，持续整局游戏：• 玩家最多装备3个遗物• 遗物品质同技能品质（白/绿/蓝/橙）• 遗物效果多样：塔属性加成、资源加成、特殊机制等• 遗物之间可产生组合效果（如同时拥有2个火焰遗物触发"烈焰之心"）

11.2 广告设计

12. 社交系统
12.1 微信社交集成
好友排行榜：基于赛季积分的周/赛季排行
分享机制：通关/获得稀有技能时可分享战绩卡片到微信群
组队挑战（后期版本）：2人合作防守模式
观战功能：可观看好友当前游戏实况（后期版本）
12.2 排行榜设计
• 周榜：每周一重置，根据本周最高挑战难度积分排名• 赛季榜：每赛季2个月，累计积分排名• 好友榜：仅显示微信好友排名• 榜单奖励：每周/每赛季发放排名奖励（钻石/头像框）
13. UI/UX设计需求
13.1 整体风格
竖屏布局，适配全面屏（18:9 ~ 21:9）
UI风格与美术风格统一，按钮带微动效反馈
核心操作区域集中在屏幕下半部分，方便单手操作
信息层级清晰：关键信息大字显示，次要信息折叠
13.2 主要界面
13.3 新手引导
• 首次进入游戏：简短世界观动画（可跳过）→ 引导建塔 → 引导释放技能• 前3波为引导关，逐步引入建造/升级/技能系统• 引导采用高亮+箭头指示，不打断玩家操作• 引导完成后可随时在设置中重新查看教程
14. 美术风格与资源需求
14.1 美术风格
推荐方案：Q版卡通像素风（2D骨骼动画）• 色彩明快饱和，适配手机小屏幕辨识度• 角色/塔/敌人设计强调剪影辨识度• 特效风格：像素化粒子+光效，避免过度复杂• UI风格：圆角卡片+渐变色，强调可点击感

16. 技术架构需求
16.1 项目架构
采用 ECS（实体-组件-系统）架构管理游戏对象
状态机管理游戏流程（主菜单→选英雄→战斗→结算）
事件驱动：模块间通过事件总线通信，降低耦合
对象池：防御塔、敌人、子弹、特效全部使用对象池
配置数据驱动：敌人/塔/技能/关卡数据用JSON配置表管理
16.2 目录结构（Cocos Creator）
assets/├── scenes/           # 场景文件├── prefabs/          # 预制体（塔、敌人、UI面板）├── scripts/│   ├── core/         # 核心系统（游戏管理器、事件总线、对象池）│   ├── battle/       # 战斗系统（塔管理、波次管理、敌人AI）│   ├── roguelike/    # 肉鸽系统（地图生成、事件系统、Build管理）│   ├── hero/         # 英雄系统（技能、成长）│   ├── ui/           # UI脚本│   ├── data/         # 数据管理（配置读取、存档）│   └── utils/        # 工具函数├── data/             # JSON配置表├── art/              # 美术资源│   ├── sprites/│   ├── spine/        # 骨骼动画（如使用）│   ├── effects/│   └── ui/├── audio/            # 音频资源└── resources/        # 动态加载资源

17. 性能优化需求
17.1 目标性能指标
17.2 优化策略
对象池：敌人/子弹/特效/金币飘字全部使用对象池，避免GC
图集合并：同界面UI资源合并为图集（Atlas），减少DrawCall
LOD/分帧：大量敌人分帧更新AI逻辑，避免单帧卡顿
资源按需加载：战斗资源延迟加载，非当前章节资源卸载
合批渲染：相同材质的节点尽量合批（Cocos自动合批 + 手动控制）
纹理压缩：使用ASTC/ETC2压缩纹理，减少显存占用
避免动态字体：使用位图字体（Bitmap Font）替代系统字体
代码分包：首包仅加载核心逻辑，其余模块按需下载
